欢迎使用 Miku 歌姬解放计划

从伴奏到歌声的本地优先工作台。无需联网，所有数据保存在本机。

  1. 1

    导入伴奏分析

    使用 tools/analyze_audio.py 或内置 librosa 分析器生成 JSON，然后在"导入本地文件"区域选择。

  2. 2

    关联无人声伴奏 WAV

    选择对应的 WAV 文件。音频只在本地使用，不会上传。

  3. 3

    选择歌词区域并填词

    在波形上拖选一段，输入中文或日文歌词。支持拼音/罗马音自动纠正。

  4. 4

    导出到歌声引擎

    支持 OpenUtau USTX 0.7、Synthesizer V 1.9.0、VOCALOID6 6.13.0、MIDI 基线四种导出格式。

找不到分析 JSON？

如果你还没有分析 JSON，可以：

  • 使用内置 librosa 分析器（仅在桌面版可用）
  • 运行 python tools/analyze_audio.py <input.wav> -o analysis.json（标准库基线）
  • 运行 python -m tools.miku_analysis.librosa_backend <input.wav> -o analysis.json（librosa 后端，精度更高）

多轨混音 · P1.2

3. 多轨伴奏混音器

非破坏编辑：原始音频永不覆盖；mute / solo / gain / pan / trim / fade 只保存参数。主输出真实生效；鼓组 / 贝斯 / 其他乐器是占位 stem，等接入音源分离后端后才会实际播放。

编辑后模式应用 trim 与 fade 包络；原始模式只保留 gain/pan/mute/solo，用于 A/B 对比。切换不记入撤销栈。

占位 stem 不影响当前试听；导入项目后混音参数会自动恢复。独奏模式下，未独奏的轨道自动静音。

音符编辑 · P1.2

4. 钢琴卷帘与音符候选

在时间轴上创建、移动、拉伸、拆分和合并音符候选；与歌词和休止共享 sample + anchor 时间模型。第一版没有真实转录后端，所有音符都是用户手工创建。

在空白区域拖出新音符；点击音符选中并编辑；拖动音符移动；拖动音符左右边缘拉伸；Shift + 点击第二个音符设为合并候选；Delete 删除选中音符；Esc 取消当前拖动；"量化"按钮把选中音符对齐到当前网格（含 1/8、三连音、附点、Swing）。

参数曲线 · P4

音符参数曲线

选中音符后，为该音符编辑音高 / 力度 / 颤音参数曲线。每个控制点是 (tick, value) 对；音高范围 -1..1（相对偏移），力度/颤音范围 0..1。

候选比较 · P4
