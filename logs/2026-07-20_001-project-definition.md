# 2026-07-20 / 001 / 建立项目定义

## 本轮目标

分析用户构想，形成第一版通用软件方法、阶段计划、Codex 接手资料和逐轮日志制度，并连接用户提供的 GitHub 仓库。

## 用户确认的要求

- 软件可导入纯音乐、伴奏或其他音乐素材。
- 软件把音频中的波形、频谱、音符状态、和弦、图形和数据等直观展示出来。
- 用户可选择一段时间并填词。
- 软件可辅助生成或衔接音调、节奏、呼吸、和声和其他参数。
- 用户可继续微调。
- 软件只提供方法和能力，不决定用户怎样创作。
- 用户举出的具体唱法只是例子，不是核心能力或产品特例。
- 每轮有效工作要生成日志，项目要保存在本地并可被其他 Codex 立即接手。
- GitHub 仓库为 `https://github.com/yelunz/MIKU-.git`。

## 决定与理由

- 采用中立项目模型和适配器架构，避免项目被单一声库厂商绑定。
- 分析、转录、生成被定义为不同能力，避免把不存在的旋律误称为可提取数据。
- 默认界面与高级数据使用渐进式呈现，兼顾小白可用性和用户自主控制。
- 自动结果必须带置信度、锁定、差异和撤销能力，避免覆盖人工工作。
