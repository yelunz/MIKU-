# 轮 002 · v0.10.1 引导页 / 示例项目加载修复

**日期**：2026-07-22
**阶段**：v0.10.0 后续修复 → v0.10.1
**目标**：用户报告 v0.10.0 安装后"没有引导页，无法导入 basic-c-major-120-v1.analysis.json"。排查发现两个根因并修复。

## 用户问题

1. **没有引导页**：启动后看不到新手引导卡片，无法点"加载示例项目"
2. **无法导入示例 JSON**：手动用文件选择器选 `basic-c-major-120-v1.analysis.json` 也失败

## 根因分析

### 根因 1：引导页未显示

- `onboarding-panel` 在 index.html 默认 `hidden`
- `onboarding.js` 的 `init()` 根据 localStorage `miku-onboarding-completed` 标记决定是否显示
- 如果用户之前测试过 v0.6.0/v0.6.1 并勾选"不再显示引导页"，localStorage 会标记 `true`，导致 v0.10.0 直接跳过引导页
- 即使 productId 改了，Electron 的 userData 路径可能因 productName 变化而不同，但 localStorage 可能因升级安装覆盖而保留旧标记
- **本质问题**：用户一旦跳过引导页，就没有任何 UI 入口重新打开引导页或加载示例项目

### 根因 2：fetch 路径错误 + webSecurity 阻止 file:// 跨目录 fetch

- `loadExampleProject()` 用相对路径 `../../fixtures/...` fetch 示例 JSON
- 打包后 app.asar 在 `win-unpacked/resources/app.asar`，fixtures 在 `win-unpacked/fixtures/`（extraFiles 的 `to: "fixtures"` 相对于安装根目录，不是 resources/）
- 从 `app.asar/web-workbench/index.html` 出发：
  - `../` = `app.asar/`
  - `../../` = `resources/`
  - `../../../` = `win-unpacked/`（安装根）← fixtures 实际位置
- 所以 `../../fixtures/` 指向 `resources/fixtures/`（错误），应该是 `../../../fixtures/`
- **更严重的问题**：即使路径对了，Electron 默认 `webSecurity=true` 会阻止 file:// 跨目录 fetch（CORS），fetch 会失败

## 修复方案

### 修复 1：导入面板内联"加载示例项目"按钮 + "重新显示引导页"按钮

**index.html**：
- 在 `.import-panel` 顶部加 `<div class="import-quick-actions">` 容器
- 内含两个按钮：
  - `load-example-inline-button`（primary）：直接调用 `loadExampleProject()`，不依赖引导页
  - `show-onboarding-button`：清除 localStorage `miku-onboarding-completed` 后重新显示引导卡片

**styles.css**：
- 新增 `.import-quick-actions` 样式（flex 布局 + primary/次要按钮区分）

**app.js**：
- `elements` 注册 `loadExampleInlineButton` + `showOnboardingButton`
- 绑定 `loadExampleInlineButton` click → 调用 `loadExampleProject()`（与引导页的 load-example-button 共用逻辑）
- 绑定 `showOnboardingButton` click → `localStorage.removeItem(ONBOARDING_KEY)` + 显示 onboarding-panel + 隐藏 import-panel + 隐藏 workbench

### 修复 2：补充 fetch 候选路径 + bridge fallback 绕过 webSecurity

**app.js `loadExampleProject()`**：
- 在 analysisCandidates 中补充 `../../../fixtures/...` 路径（Electron 打包后正确路径）
- 新增 bridge fallback：fetch 全部失败时，用 `bridge.resolvePackagedFixture(relPath)` 获取绝对路径，再用 `bridge.readFileAsText(absPath)` 读取 JSON
- 这绕过了 Electron webSecurity 对 file:// 跨目录 fetch 的限制（Node fs 不受 CORS 约束）

**app.js `tryAssociateExampleAudio()`**：
- 同样补充 `../../../fixtures/...` 路径
- 新增 bridge fallback：用 `bridge.resolvePackagedFixture` + `bridge.readFileAsArrayBuffer` 读取 WAV

### 修复 3：main.js 新增 `miku:resolvePackagedFixture` IPC handler

**main.js**：
- 新增 `ipcMain.handle("miku:resolvePackagedFixture", ...)` 
- 输入：相对 fixtures 目录的子路径（如 `basic-c-major-120-v1/librosa-analysis-v2.json`）
- 查找顺序：
  1. `path.join(path.dirname(process.resourcesPath), "fixtures", relativePath)`（打包后：`<安装根>/fixtures/`）
  2. `path.join(__dirname, "..", "..", "fixtures", relativePath)`（开发模式）
  3. `path.join(__dirname, "..", "fixtures", relativePath)`（备用）
- 安全：拒绝 `..` 和绝对路径（防路径穿越）
- 返回找到的绝对路径字符串；找不到返回空字符串

**preload.js**：
- bridge 新增 `async resolvePackagedFixture(relativePath)` 方法
- 调用 `ipcRenderer.invoke("miku:resolvePackagedFixture", relativePath)`

## 用户升级指南

1. 卸载 v0.10.0（或直接覆盖安装 v0.10.1）
2. 安装 `Miku歌姬解放计划-0.10.1-win-x64.exe`
3. 启动后会看到导入面板顶部有两个按钮：
   - **"加载示例项目（推荐首次使用）"**：一键加载内置 C 大调 120 BPM 50 秒示例
   - **"重新显示引导页"**：清除 localStorage 后重新显示新手引导卡片
4. 点"加载示例项目"即可加载 `basic-c-major-120-v1` 示例（无需手动找文件）

## 决定

1. **不依赖引导页加载示例**：导入面板内联按钮是永久入口，即使用户跳过引导页也能加载示例。引导页只是首屏介绍，不再是加载示例的唯一路径。
2. **bridge fallback 优先于 fetch**：Electron webSecurity 对 file:// 跨目录 fetch 的限制是设计决策（安全），不应关闭。用 Node fs 通过 IPC 读取是更可靠的方式。fetch 仍作为第一尝试（浏览器模式下 fetch 是唯一方式）。
3. **PyInstaller 不重新打包**：v0.10.1 只改了前端和 Electron 主进程 IPC，分析服务后端无变化，复用 v0.10.0 的 PyInstaller 产物（288.13 MB）。

## 未决问题

1. **真实 Electron 回归测试**：本轮修复未在真实 Electron 中验证"点加载示例项目按钮 → bridge.resolvePackagedFixture → readFileAsText → applyAnalysis"完整链路。用户测试时请重点验证。
2. **引导页为何未显示**：可能是 localStorage 旧标记，也可能是 onboarding.js 初始化问题。v0.10.1 加了"重新显示引导页"按钮作为 workaround，用户可手动触发。如果按钮也不显示，说明是更深的 DOM/CSS 问题。
3. **macOS / Linux 打包**：仍然延后。
