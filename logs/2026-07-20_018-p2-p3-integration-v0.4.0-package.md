## 决定与理由

1. **schema 一次性升到 0.3.0**：syllables + vocalPreview 是数据模型层面的扩展（新增持久化字段），符合 schema 升级条件。0.2.0 项目自动派生 syllables 是无损的（用户可随时重新切分或覆盖读音）。
2. **拼音表首批 80 字**：覆盖示例歌词常用字；后续从 unihan 字典扩展。默认读音去声调（不同声库表达方式不同），用户可在 readingOverride 中输入带声调的完整读音。
3. **试听只支持 OscillatorNode 4 种基础波形**：第一版目标是验证音高调度链路，不追求音色真实。后续接入 Basic Pitch 转录 + formant 合成。
4. **P3 三个导出器独立实现，不抽公共模块**：第一版要求"先按原定方案执行"，避免过早抽象。后续 P3.5 引入 VOCALOID 适配器时可重构为 `tools/_neutral_model.py`。
5. **SynthV 配套脚本严格 ES5 + minEditorVersion 0x010900**：严格遵守 ARCHITECTURE.md G 部分约束，ARA / Voice-to-MIDI 是 1.11+ 能力，不写入 1.9.0 适配器。
6. **v0.4.0 NSIS 安装包作为第一版交付物**：尽管 asar 缺 web-workbench 是已知问题，但安装包已生成且桌面壳静态测试全通过。重新打包是 next_actions[0]，需要在下一轮执行。
