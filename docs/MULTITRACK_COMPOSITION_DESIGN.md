# 多轨伴奏与歌声编曲重构

## 用户反馈与产品纠正

当前 Web 原型能够播放伴奏、显示波形和分析候选、框选歌词区域，但仍然更像“分析结果查看器”，不像可以实际编曲的虚拟歌姬编辑器。用户明确指出：

- 框选缺乏可靠吸附，相邻歌词区域会出现无意义的小缝。
- 播放/暂停缺少空格快捷键，编辑手感不符合常见音频软件习惯。
- 只显示波形、段落和和弦，不足以逐音符调整音高、时值、正拍/反拍、休止和节奏。
- 伴奏应尽可能拆成多个可观察、可静音、可调整的声部，并把可转录内容显示成可编辑音符候选。
- 完全不懂乐理的用户也应该能通过听觉、形状和自然语言体验编曲，而不是先学习钢琴卷帘术语。

这些属于产品能力和体验要求，不是某一种唱法或审美示例。P1 的目标需要从“显示伴奏分析”修正为“可可靠操作的多轨编曲工作台”。

## 三个不能混为一谈的过程

### 1. 音源分离（Source Separation）

把混合伴奏估计为鼓、贝斯、其他乐器等音频 stem。输出仍是音频，不是 MIDI。用户可以对 stem 静音、独奏、调音量、声像、裁切、时间伸缩或替换，但模型可能产生串音和伪影，也不能恢复录音时真实存在的原始分轨。

### 2. 音频转录（Audio-to-Symbolic Transcription）

在原音频或分离 stem 上估计起音、鼓点、音高、时值、力度和弯音，输出带置信度的 `NoteEvent` 或 `DrumEvent`。这些事件可以进入钢琴卷帘，但会漏音、重音、错八度、错延音；转录 MIDI 的修改不会自动改变原 WAV，必须用音源重合成或对分离 stem 做对应处理才能听见变化。

### 3. 歌声旋律生成/输入

无人声伴奏中不存在唯一的人声旋律答案。软件可以根据节拍、和声、音域和用户约束生成候选，也允许用户手工输入；结果属于独立 `VocalTrack`，不能冒充从伴奏中提取出的既有旋律。

## 新的多轨工作流

```text
不可变原始伴奏
  ├─ 快速分析：节拍 / 小节候选 / 起音 / 调性 / 和弦 / 段落
  ├─ 可选音源分离：drums / bass / vocals / other 或其他可用 stem
  │    ├─ 鼓点候选：kick / snare / hat / other onset
  │    ├─ 贝斯音符候选
  │    └─ 和声/乐器音符候选
  ├─ 非破坏伴奏编辑：mute / solo / gain / pan / cut / warp / effect
  └─ 用户确认的共享时间网格
          ↓
歌词句子轨 → 字/假名/音节 → 演唱音符候选 → 主唱/和声轨 → 参数曲线 → 外部适配器
```

所有自动结果都是可替换分析层。用户值、分析原值、手工修正、锁定状态和修改历史必须分开保存。

## 时间与边界模型

专业编辑不能只保存相互独立的浮点秒数。内部同时保存：

- `sample`：音频采样位置，是音频定位的权威基准。
- `tick`：音乐刻度，首版采用 PPQ 960，用于节拍、量化、音符和 MIDI。
- `Anchor`：相邻段落、歌词、休止或音符共享的边界 ID。

连续区域采用半开区间：

```text
Lyric A [anchor-1, anchor-2)
Lyric B [anchor-2, anchor-3)
```

移动 `anchor-2` 是一次事务，同时改变 A 的结束和 B 的开始，因此数据层不会产生漏缝。小于等于当前吸附单位的未知缝自动闭合；更大的空段显示为“未分配”并允许转为明确 `RestEvent`。换气是点事件或短事件，不和休止混为一类。

## 轨道与事件模型

- `AudioAsset`：不可变原始文件、哈希、采样率、声道和授权来源。
- `AnalysisRun`：引擎、版本、模型哈希、参数、设备、输入哈希、状态和警告。
- `StemAsset`：角色、缓存路径/哈希、偏移采样、重构残差和串音/削波启发指标。没有校准概率时不得伪造置信度。
- `TempoMap`：拍点、下拍、小节、拍号与变速段；每个事件同时保存 sample、tick、来源和置信度。
- `NoteEvent`：stem、start/end sample、start/duration tick、浮点音高、力度、弯音、置信度、替代候选、锁定和手工覆盖。
- `DrumEvent`：起音、类别候选、力度和置信度；鼓轨不使用旋律音高转录器硬套 MIDI 音高。
- `HarmonyEvent`：根音、类型、低音、候选集合和算法分数类型。
- `LyricRegion`：共享 anchor、语言、原文、读音、音节/假名拍和音素。
- `RestEvent`、`BreathEvent`：显式表示休止和换气。
- `VocalTrack` / `VocalPhrase`：独立于伴奏转录的演唱数据。
- `EditGraph`：非破坏音频操作、符号事件修改、撤销、重做、差异和局部锁定。

## 面向音乐小白的渐进界面

简洁视图和专业视图必须使用同一份数据，只改变表达密度。

| 层级 | 用户看到什么 | 用户如何操作 | 专业数据 |
|---|---|---|---|
| 0 听与选 | 整曲地图、强弱、段落、播放头 | 播放、框选、循环 | sample、秒、段落候选 |
| 1 形状编曲 | “高/低、密/疏、强/弱、留白” | 整句抬高、压低、提前、拖后、加停顿 | 音符、tick、力度、休止 |
| 2 句子与节奏块 | 歌词句、字/假名块、脚步式节拍格 | 拆分、合并、拉长、缩短、放到两拍之间 | 音节边界、量化、反拍 |
| 3 音符编辑 | 高低块、时值、主唱/和声平行轨 | 移动、拉伸、多选、复制、候选替换 | MIDI 音高、tick、声部 |
| 4 专业模式 | 钢琴卷帘、音高/力度/发音曲线 | 精确输入和参数编辑 | 音素与引擎字段 |

自然语言只描述变化，不判断审美：例如“整体压低”“结尾上扬”“更密”“更疏”“这里留空”“添加上方声部”。软件显示修改前后差异，不使用“最好”“正确唱法”或“最适合”。

术语采用先听后解释：

- 反拍：先显示“声音落在主要脚步之间”，再展开术语和精确 tick。
- 休止：显示明确透明空段，并写“这里没有演唱，不是漏掉内容”。
- 延音：音节或音符末端提供尾巴手柄，同时显示跨越几个节拍。
- 声部：先称“另一条同时演唱的线”，再展开上声部、下声部和同度叠唱。

## 伴奏编辑边界

第一版采用非破坏编辑：原始音频永不覆盖；裁切、音量、声像、静音、独奏、时间伸缩、warp marker 和效果只保存参数，预览缓存可重建。

用户编辑转录得到的音符时，界面必须说明当前试听来源：

- “原始/分离音频”：音符修改只改变标注，不改变声音。
- “重合成候选”：使用选定音源播放修改后的符号事件。
- “外部歌声/乐器引擎”：通过适配器渲染。

不能让用户误以为移动 MIDI 方块已经无损修改原始混音。

## 技术验证候选与许可边界

以下只作为技术 spike 候选，不等于已经引入依赖：

- `librosa`：ISC 许可，用于节拍、起音、色谱和 HPSS 快速分析；下拍/拍号仍需单独验证。[官方仓库](https://github.com/librosa/librosa)
- `Basic Pitch`：Apache-2.0，可输出音符、MIDI 和 pitch bend；官方明确最适合一次处理单一乐器，因此优先用于分离后的 bass/乐器 stem，不直接承诺完整伴奏转录准确。[官方仓库](https://github.com/spotify/basic-pitch)
- `Demucs v4`：可作为 4-stem 质量基准，但原仓库已归档，预训练权重许可问题仍未明确；现阶段不得把权重随发行包分发，只做可选实验后端和许可核查。[官方仓库](https://github.com/facebookresearch/demucs)、[权重许可问题](https://github.com/facebookresearch/demucs/issues/327)
- `Essentia`：算法覆盖广，但核心 AGPLv3、模型含非商业或专有许可；没有商业许可时不作为默认可分发核心。[许可说明](https://essentia.upf.edu/licensing_information.html)
- `SoundTouch`：LGPL-2.1，可验证第一版实时速度/音高预听；分发时必须履行 LGPL 或取得商业许可。[官方页面](https://surina.net/soundtouch/index.html)
- `Rubber Band`：GPL/商业双许可；闭源发行需要商业许可，不作为未经审查的默认依赖。[许可说明](https://breakfastquay.com/rubberband/license.html)

## 实施顺序

### P1.1 编辑器手感

- 空格播放、点击定位、可靠拖选、边缘手柄、吸附、共享边界、显式休止、缩放锚点和键盘微调。
- 用当前 fixture 验证 120 BPM 下 1/2 拍严格为 0.25 秒，相邻歌词共享边界。

### P1.2 多轨与音符工作台

- 建立 sample + PPQ + Anchor 数据模型。
- 增加伴奏 stem 轨、鼓点轨、贝斯/和声音符候选轨和可编辑钢琴卷帘。
- 支持音符创建、移动、拉伸、拆分、合并、多选、量化、反拍、三连音、附点和 Swing。
- 加入非破坏 mute/solo/gain/pan/cut 与原始/重合成试听切换。

### P1.3 分析后端对比

- `librosa` 快速层与现有标准库基线 A/B。
- `Basic Pitch` 仅在单乐器/分离 stem 上验证准确率、速度和模型分发。
- `Demucs` 做 4-stem 质量、CPU/GPU、内存、模型体积、权重许可和三平台 spike。
- 另行比较下拍/拍号后端，未通过前不把小节线当确定事实。

### P2 歌声垂直切片

- 中文/日文读音、音节/假名拍、演唱候选、主唱/和声轨、A/B、撤销、锁定和局部重生成。
- 首个端到端导出仍优先 OpenUtau USTX 0.6，然后是 Synthesizer V Studio Pro 1.9.0；VOCALOID6 6.13.0 保持后续目标。

## 验收方向

- 连续按空格不会滚动页面或在输入歌词时误触播放。
- 框选和边缘移动可以吸附 1 拍、1/2 拍、1/4 拍；Alt/Option 可临时绕过。
- 两段歌词共享同一个 anchor，移动边界后数据和像素均无漏缝；大空段显示为休止/未分配。
- 音符候选可以逐个移动、拉伸、拆分、合并和量化，并区分正拍、反拍和休止。
- 每个 stem 可独奏/静音，转录事件和原音频状态清楚分开。
- 完全不懂乐理的测试者能在 10 分钟内完成导入、试听、填词、产生候选、三种修改、A/B、撤销和保存，并能用自己的话说明改变了高低、时间、强弱或留白中的什么。
- 用户可以拒绝所有建议、手工输入或采用任意反差方案；验收不以某种审美结果为标准。
