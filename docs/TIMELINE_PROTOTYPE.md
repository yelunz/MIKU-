# 时间轴交互原型

## 目的

第一版交互原型用于验证“看见伴奏 → 选择区域 → 输入歌词 → 后续生成可编辑演唱草稿”的信息结构。它不是已经完成的桌面应用，也不把测试夹具真值伪装成算法准确率。

本线框图的可运行后继版本位于 `prototype/web-workbench/`。它已经读取真实分析 JSON 和本地 WAV，并实现播放、选区、歌词区域、和弦修正和项目往返；本文件继续记录不依赖具体实现的交互原则。

## 首屏轨道

从上到下采用同一条时间坐标：

1. 秒与小节标尺。缩小时突出小节线，放大后再显示每拍线。
2. 段落候选。点击段落可快速建立选区，边界必须可纠正。
3. 和弦候选。每个块保存范围、候选、置信度、来源和恢复值。
4. 波形与能量总览。节拍网格叠加显示，频谱默认不挤入同一轨。
5. 歌词区域。歌词独立于分析结果，可拖动、拆分、合并和锁定。
6. 演唱草稿。无人声伴奏导入后保持“尚未生成”，不能写成已识别主旋律。

默认开启波形、能量、段落、和弦和按缩放级别显示的节拍网格；频谱、频段、算法参数和精确数据放在可展开的高级层。

## 交互规则

- 用户可以拖选任意范围，也可以点击段落或和弦块快捷选择。
- 选区与播放头是两个独立状态；单击定位播放，拖拽创建选区。
- 默认吸附小节或节拍，并提供临时取消吸附和精确数值输入。
- 中文 9–25 秒和日文 25–41 秒只标为“测试选区”，语言由用户选择，不从伴奏猜测。
- 输入歌词只建立 `LyricRegion`；生成音符、呼吸、和声和参数属于之后可单独运行、撤销和锁定的操作。
- 自动值统一使用“分析结果（可修正）”或“候选”，用户确认后标记“手动确认”，并可恢复原分析值。

## 渐进呈现

基础层先用自然语言说明结构、强弱、和弦与下一步动作。选择对象后再展开秒、拍位、音名、MIDI、置信度、算法版本和参数。基础视图和高级视图必须读取同一份数据，不能形成两套互相冲突的状态。

## 禁止误导的文案

- 不写“准确和弦”，写“和弦候选”。
- 不写“已识别主旋律”，写“候选旋律”或“尚未生成演唱草稿”。
- 不写“最适合的唱法”，写“按当前约束生成候选”。
- 不写“检测呼吸”，写“生成/编辑换气建议”。
- 不写“一键完成整首歌”，写“为选区生成可编辑草稿”。

## 尚待原生原型验证

- 波形/频谱长音频缩放与拖动性能。
- 鼠标、触控板与键盘的选区手柄交互。
- 播放头、循环区和歌词区重叠时的视觉优先级。
- 分析差异预览、锁定和恢复分析值的完整流程。
