# 阶段计划

计划以“通过验收条件”推进，不把尚未验证的技术假设写成日期承诺。

| 阶段 | 目标 | 主要产出 | 通过条件 |
|---|---|---|---|
| P0 产品定义 | 明确首批输入、输出和交互边界 | 产品定义、架构、线框图、测试素材规范 | 三系统、首批适配器、语言与无人声伴奏测试范围被记录，剩余版本细节可验证 |
| P1 音频工作台原型 | 证明导入、播放、选区和多层显示可用 | 桌面原型、波形、频谱、节拍/小节、基础项目保存 | 一段测试音频可流畅缩放、播放和选区，分析层可开关和修正 |
| P2 歌词与歌声垂直切片 | 跑通“选区—填词—歌声草案” | 中文/日文歌词区域、读音纠正、切分与对齐、音符候选、基础试听、撤销与锁定 | 用户可分别用中文和日文生成并修改一小段演唱草案，重生成不覆盖锁定内容 |
| P3 首批引擎适配 | 将中立项目可靠交给首批歌声后端 | MIDI 基线、OpenUtau USTX 0.6 适配器、Synthesizer V Studio Pro 1.9.0 适配器、字段损失报告 | 导出后在两个第一优先目标编辑器打开，音符、歌词和节奏达到约定一致性 |
| P3.5 VOCALOID 适配 | 接入 VOCALOID6 Editor 完整版 6.13.0 | `vocaloid/6/standalone` 主适配器、旧版 MIDI 降级矩阵和字段损失报告 | 交换文件能在 6.13.0 完整版打开并通过一致性测试；Linux 完成文件生成与检查 |
| P4 完整编排能力 | 扩展呼吸、和声和参数编辑 | 呼吸编辑、和声轨、参数曲线、候选比较 | 各能力可独立运行、独立撤销并在局部锁定下重新生成 |
| P5 可用性与发布 | 让非专业用户可以独立完成任务 | 新手引导、错误恢复、安装包、示例项目、性能和兼容测试 | 新用户不查乐理资料即可完成规定的端到端测试任务 |

## P0 已确认内容

1. 核心应用支持 Windows、macOS、Linux。
2. 首批适配用户现有的 Synthesizer V Studio Pro 1.9.0 和验证时最新版稳定版 OpenUtau。
3. VOCALOID6 Editor 完整版 6.13.0（Standalone）进入后续适配范围，旧版使用 MIDI 降级路径。
4. 首版支持中文、日文歌词，暂不支持英文。
5. 第一轮使用无人声伴奏。

## P0 剩余确认与验证

1. MVP 必须显示的分析层及默认开关组合。
2. 中文读音纠正和日文原文/假名输入流程。
3. 桌面技术栈的三平台构建与音频性能验证。
4. VOCALOID6 6.13.0 完整版真实测试环境的可获得性。

## 下一轮建议工作

项目自有 50 秒测试伴奏、标准答案和交互时间轴原型已经完成。下一轮用真实桌面技术栈接入当前分析 JSON，并继续验证：

- 首屏波形生成时间。
- 节拍、小节、和弦和段落分析的准确性及耗时。
- 时间轴缩放和拖动性能。
- 分析结果的人工纠正流程。
- 项目保存、关闭和恢复的一致性。
- Windows、macOS、Linux 的分析结果一致性与打包差异。

## P1 技术验证进展

- 已完成 50 秒确定性无人声伴奏夹具和可运行的标准库分析 CLI。
- 已完成可交互时间轴原型，验证段落、和弦、波形/能量、选区、歌词区域和演唱草稿空状态的信息结构。
- 已完成零依赖 Web 工作台：读取真实分析 JSON、关联本地 WAV、播放/定位、选区、中文/日文歌词、和弦修正和项目往返已在 Windows Edge 烟测通过。
- 已建立浏览器 `DesktopBridge` 边界；Electron 43.x 为首选验证候选，Tauri 2.11.x 保留为备选，尚未正式锁栈。
- 当前分析基线命中 119.993 BPM、0.970 秒首拍和 C major；9/25/41 秒主段落边界均进入候选。
- 段落仍有额外边界，和弦严格 Top-1 检查为 0.875，低于 0.9 验收线；P1 尚未完成。
- 下一步把当前工作台封装进最小 Electron 验证壳，接入 Python 分析进程并建立三平台 packaged artifact；同时继续对比生产级分析库。

## 重要风险与处理

| 风险 | 影响 | 处理方式 |
|---|---|---|
| 混合音频分析不是确定答案 | 和弦、段落或音符可能错误 | 显示置信度、允许纠正、保留原值和多候选 |
| 不同输入类型被混为一谈 | 用户期待提取不存在的旋律 | 导入时确认模式，界面区分“分析/转录/生成” |
| 外部格式丢失参数 | 导出后效果变化 | 每个适配器维护能力表，导出前生成损失报告 |
| OpenUtau 或外部编辑器版本漂移 | 原有适配器在升级后失效 | 验证时固定稳定版号和格式版本，建立三平台回归样例 |
| VOCALOID 各代格式和能力不同 | 一个笼统适配器会误报兼容性 | 按主版本与编辑器形态拆分能力矩阵，版本未知时只承诺 MIDI 基线与损失报告 |
| 中文/日文读音与声库方案不同 | 相同歌词在不同声库发音不一致 | 分离原文、标准读音和引擎音素，导出时检查声库与 Phonemizer 依赖 |
| 自动生成覆盖人工工作 | 用户失去控制和信任 | 字段级锁定、差异预览、撤销与版本历史 |
| 新手界面过简或专业界面过载 | 一边不可控，另一边难使用 | 渐进式呈现，同一数据提供简洁视图和高级视图 |
| 音频、歌曲和声库许可不清 | 无法合法分发或演示 | 只使用有授权的测试素材，不把第三方内容提交仓库 |
| 工作名称涉及现有角色名称 | 公开或商业发布存在权利风险 | 发布前完成商标和角色使用审查，必要时改名 |
