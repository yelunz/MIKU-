# 产品定义

## 一句话定义

把本地音频转换成可读、可选、可填词、可生成歌声编排并可导出到不同歌声引擎的图形化工作台。

## 产品职责与用户职责

软件的职责是提供能力：

- 读取音频并提取可观察的数据。
- 用直观图形呈现数据，同时保留精确数值。
- 把歌词与时间范围、节拍和音符建立关联。
- 自动生成或辅助生成演唱所需的数据。
- 允许用户设置约束、修改结果、锁定局部和重新生成。
- 把中立的内部数据转换成外部歌声软件能够接收的格式。

用户的职责是决定作品要表达什么、使用哪些能力、接受或拒绝哪些建议，以及最后采用什么效果。软件不内置唯一正确的唱法或创作路线。

## 首版已确认范围

- 核心应用运行于 Windows、macOS、Linux。
- 第一优先适配用户现有的 Synthesizer V Studio Pro 1.9.0 与验证时最新版稳定版 OpenUtau。
- VOCALOID6 Editor 完整版 6.13.0（Standalone）列为后续主适配目标；旧版本提供带损失报告的 MIDI 降级兼容。
- 歌词语言支持中文和日文，暂不支持英文。
- 第一轮技术验证使用无人声伴奏。用户暂时没有可提供的伴奏，因此使用项目自行合成、可重复生成且带标准答案的测试夹具。

“三平台运行”约束核心应用本身；外部适配器还必须检查对应歌声编辑器在该操作系统上的安装与支持状态。

VOCALOID 官方编辑器没有 Linux 版，因此 Linux 验收范围是核心应用与交换文件的生成/检查，不包含启动官方编辑器。VOCALOID6 6.13.0 完整版的真实打开验收在 Windows/macOS 进行。

## 已确认的能力范围

### 1. 音频导入

支持导入常见本地音频。产品需要识别并明确区分至少三种输入情况：

- 已有主旋律的纯音乐：可以尝试转录主旋律。
- 无人声伴奏：可以分析节奏、和声和结构，但演唱旋律需要另行生成或由用户输入。这是第一轮技术验证的输入类型。
- 完整歌曲：在用户合法拥有处理权的前提下，可选地进行人声分离和转录。

输入类型可以由用户选择，也可以由软件建议，但不能静默假设。

### 2. 音频分析与可视化

分层展示，避免把所有专业信息一次塞给新手：

- 总览层：波形、时长、能量变化、段落边界。
- 音乐层：速度、拍号、小节、节拍、调性、和弦、候选主旋律。
- 声学层：频谱、频段能量、瞬态和其他可解释特征。
- 数据层：精确时间、音名、MIDI 音高、置信度、算法来源。

每项自动分析数据都要支持显示置信度、手工纠正和恢复原分析值。

### 3. 区域选择与歌词编辑

- 用户可在时间轴上选中任意区域。
- 可给区域输入整句歌词，再拆成字、音节或音素。
- 可调整句子起止、字词边界、延音、停顿和换气点。
- 歌词与音符的对齐是独立、可编辑的数据，不应写死在音频分析结果里。
- 中文管线至少需要保存汉字原文、可修正读音和目标声库音素；不能静默猜测多音字。
- 日文管线至少需要保存原文、可修正假名读音、拍和目标声库音素。
- 内部数据模型要把显示文字、标准读音和引擎音素分开，避免绑定某个声库的发音系统。
- 英文切分、读音和音素转换不进入首版实现或验收。

### 4. 歌声编排

对所选区域提供可单独启用的辅助能力：

- 音符或旋律候选。
- 节奏与歌词对齐候选。
- 音域约束与移调。
- 呼吸位置和句间空隙。
- 和声声部。
- 音高曲线、力度、发音时序及目标引擎支持的其他参数。

生成结果必须说明它修改了哪些数据。用户可以锁定任何一类或一个区域，只重新生成未锁定部分。

### 5. 微调与比较

- 基础视图用图形、拖拽和少量通用控件完成修改。
- 高级视图显示钢琴卷帘、参数曲线和精确数据。
- 支持撤销/重做、修改前后试听、多个候选比较和局部回退。
- 任何用户约束都应成为明确的数据，而不是只存在于一次性的自然语言提示中。

### 6. 保存与导出

- 保存中立的本地项目，保留音频引用、分析结果、歌词、歌声轨、锁定状态、历史和适配器设置。
- 基线导出格式为标准 MIDI；具体歌声软件通过独立适配器接入。
- 导出前显示目标格式会保留和丢失哪些数据。

## 第一版不承诺的内容

- 不承诺从任意混合音频中百分之百正确识别和弦、段落或音符。
- 不承诺某个商业歌声软件的全部私有参数都能无损互转。
- 不替用户取得歌曲、伴奏、角色、声库或模型的使用权。
- 不用单一“生成整首歌”按钮取代可观察、可修改的制作过程。

## 第一版体验验收

给定一段合法的无人声伴奏，用户能在同一工作台完成：导入、看见时间轴和基础分析、选择片段、输入中文或日文歌词、得到一条可编辑的演唱草案、修改至少三类数据、保存本地项目并导出到 OpenUtau 或 Synthesizer V Studio Pro 1.9.0 的首批适配路径。核心流程需要在 Windows、macOS、Linux 分别通过验收；外部后端集成深度按对应系统能力矩阵验收。OpenUtau USTX 0.6 是首个端到端工程导出验收，VOCALOID6 Editor 6.13.0 完整版在后续适配阶段验收。
